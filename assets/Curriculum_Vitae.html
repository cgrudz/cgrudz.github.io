Software and Educational Products

DAPPER

I am a contributor to the Nansen Center Data Assimilation Package in Python for Experimental Research (DAPPER). DAPPER is a set of templates for benchmarking the performance of data assimilation (DA) methods. The software provides experimental support and tutorials on DA.

Numerical benchmarks for iterative ensemble / ensemble-variational smoothers

This code repository contains research code studying the effects of model stochasticity and nonlinearity on the performance on different formulations of the iterative, ensemble Kalman smoother for joint state-parameter estimation in the stochastically forced Lorenz-96 model.

Numerical benchmarks for the stochastically forced Lorenz-96 model

This is the code repository for the manuscript “On the numerical integration of the Lorenz-96 model, with additive noise, for benchmark twin experiments” This includes the code for the novel strong order 2.0 Taylor scheme derived for the L96-s system.

Electric grid model reduction

This is the code repository for the manuscript “Structure-and physics-preserving reductions of power grid models” This includes the code for the reduction and interactive visualization of a real power grid network, from a Midwestern utility.

Mathematical Theory of Data Assimilation with Applications

Lectures given in the London Mathematical Society 2019 Summer Research School on the Mathematics of Climate. This introduces the fundamentals of data assimilation and estimation theory from a Bayesian perspective to new users, with an emphasis on intuition behind the methods. These lectures were developed in collaboration with Alberto Carrassi, Patrick Raanes and Marc Bocquet.

ADA compliant teaching workflow

This workflow has been designed to meet as best as possible ADA web accessibility stabdards and to produce mathematical typesetting content that is fully accessible for teaching. Templates are provided to create persistent, accessible HTML documents that can be printed identically for phyisical copies. Presentation templates are included for RMarkdown-based mathematical and software lectures.

Lecturer

University of Nevada, Reno

STAT 757
Applied Regression Analysis
This course introduces fundamental techniques of applied regression within a frequentist perspective. Students perform data analysis, construct regression models, quantify model uncertainty and qualify their analysis by evaluating model assumptions with diagnostics. Mathematical theory is introduced as needed, but the emphasis is on the selection, application and evaluation of basic statistical regression models.

STAT 445 / 645
Introduction to statistical computing
This course introduces computational statistical methods within a modern computing environment. Topics include programming in R, univariate statistical analysis, multivariate distributions, numerical linear algebra, numerical integration and numerical differentiation, root finding and minimization/maximization of objective functions. Basic concepts in regression, maximum likelihood estimation and Markov chain Monte Carlo techniques are introduced.

STAT 152
Introduction to Statistics
This course introduces the theory and practice of probability and statistics emphasizing their language, essential ideas, and concepts. Topics include the foundations of probability theory, basic descriptive statistics, graphical representation of data, point and interval estimation, hypothesis testing, correlation and regression analyses. The statistical package StatCrunch is used to illustrate the concepts discussed in class using real data sets.

Calculus II

MATH 190
Topics in Mathematics: A Climate of Uncertainty
This freshman seminar introduces mathematical concepts at the basis of numerical weather prediction and modeling the global climate at a high level. Students run experiments in the education EdGCM to explore physical concepts in the climate process introduced in class. Students complete a final modeling project in which they develop their own climate change mitigation proposal and simulate the effects of their proposal versus business-as-usual scenario.
